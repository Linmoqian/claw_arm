## 什么时候使用这个技能？

当用户的意图符合以下**任意一条**时，应激活本技能：

- 用户说"帮我拿xx"、"把xx抓过来"、"抓取桌上的xx"
- 用户要求机械臂去"找到"某个物体并抓取
- 用户想知道摄像头看到了什么物体、物体在哪
- 用户需要先识别再操作的视觉引导任务
- 上游任务（如倒水）需要先定位杯子位置

**不要使用本技能的场景**：
- 用户已经给出了精确的关节角度或坐标 → 直接用 `arm-control`
- 纯语言理解任务，不涉及视觉 → 交给 LLM

## 描述

`vision-grasp` 是一个视觉引导的抓取技能，完整流程为：

```
摄像头采集图像
    ↓
目标检测（YOLOv8 / 大模型视觉 API）
    ↓
像素坐标 → 3D 空间坐标（相机标定 + 深度估计）
    ↓
生成抓取位姿（预抓取位 → 抓取位）
    ↓
调用 arm-control 执行运动
```

### 支持的检测后端

| 后端 | 适用场景 | 是否需要 GPU |
|------|----------|-------------|
| YOLOv8 (本地) | 已知物体类别，低延迟 | 推荐但非必须 |
| 通义千问视觉 API | 开放词汇，任意物体描述 | 否（云端推理） |
| OpenCV 颜色/形状 | 简单物体，无需模型 | 否 |

### 摄像头配置

| 摄像头 | 分辨率 | 用途 |
|--------|--------|------|
| USB 摄像头 (high) | 1280×720 | 全局俯视，场景理解 |

## 参数说明

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| target | str | 必填 | 要抓取的目标描述，如 "水杯"、"红色方块" |
| camera | str | "high" | 使用的摄像头："high" |
| detector | str | "yolo" | 检测后端："yolo"、"qwen_vl"、"color" |
| arm_port | str | "COM7" | SO100 机械臂串口端口 |
| confidence | float | 0.5 | 检测置信度阈值 (0.0-1.0) |
| grasp_height | float | 0.05 | 抓取接近高度 (米)，物体上方多高开始下降 |
| auto_execute | bool | true | 检测到后是否自动执行抓取 |
| device_id | int | 0 | 摄像头设备 ID（OpenCV） |
| show_preview | bool | false | 是否显示检测可视化窗口 |

## 权限要求

- 摄像头访问权限（USB 摄像头）
- 网络访问（如使用通义千问视觉 API）
- USB 串口访问（调用 arm-control 时需要）
- Python 3.10+

## 依赖

**必需**：
- opencv-python（图像采集与处理）
- numpy（坐标计算）

**按检测后端选装**：
- ultralytics（YOLOv8 本地检测）
- openai 或 dashscope（通义千问视觉 API）

**抓取执行**：
- arm-control 技能（同级 skill，基于 LeRobot SO100Follower 控制 SO100 机械臂）

## 实现细节提示

<details>
<summary>坐标变换流程（点击展开）</summary>

1. 检测器输出像素坐标 `(u, v)` + 边界框 `(x1, y1, x2, y2)`
2. 根据相机内参矩阵 K 和深度值 d 计算相机坐标系下 3D 点：
   - `X_cam = (u - cx) * d / fx`
   - `Y_cam = (v - cy) * d / fy`
   - `Z_cam = d`
3. 通过手眼标定矩阵 T_base_cam 转换到机械臂基座标系
4. 生成预抓取位姿（目标上方 grasp_height 处）和抓取位姿

</details>

<details>
<summary>与 arm-control 的调用关系（点击展开）</summary>

本技能不直接控制舵机，而是：
1. 完成视觉检测和坐标计算
2. 将目标位姿转换为关节角度（逆运动学）
3. 调用 `arm-control` 的 `set_joint_positions` 和 `set_gripper` 接口

这种分层设计保证了技能的可组合性。

</details>

<details>
<summary>错误处理策略（点击展开）</summary>

| 错误 | 处理 |
|------|------|
| 未检测到目标 | 重试 3 次，换摄像头，报告用户 |
| 置信度过低 | 提示用户确认 |
| 逆运动学无解 | 目标超出工作空间，报告用户 |
| 抓取失败 | 通过夹爪力反馈判断，重试或报告 |

</details>
