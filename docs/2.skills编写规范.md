Gemini 说
要自定义 OpenClaw 的 Skill 并使其被识别和调用，可以按照以下步骤进行。OpenClaw 的 Skill 遵循 AgentSkills 规范，本质上是一个包含元数据和说明文件的文件夹。

1. 创建 Skill 文件夹
首先，你需要决定 Skill 的存放位置。OpenClaw 会从以下路径加载 Skill（优先级从高到低）：

当前工作区：<workspace>/skills/（仅对该工作区 Agent 可见）。

用户目录（推荐）：~/.openclaw/skills/（对所有 Agent 可见）。

自定义目录：可以在配置文件中指定任何路径。

操作： 创建一个新文件夹，例如 my-custom-skill。

2. 编写 SKILL.md (核心文件)
在文件夹内创建 SKILL.md 文件。这是 OpenClaw 识别 Skill 的关键。它由 YAML Frontmatter（元数据）和 Instructions（指令/文档）组成。

Markdown
---
name: my-custom-skill
description: 这是一个自定义技能，用于执行特定任务（如查询本地数据）。
metadata: { "openclaw": { "emoji": "🛠️", "requires": { "bins": ["python3"] } } }
---

# 技能说明
当用户要求 [描述触发场景] 时，你可以调用此技能。

## 使用方法
你可以通过执行以下命令来调用此技能：
`python3 {baseDir}/main.py --query "用户的问题"`

{baseDir} 会自动替换为该 Skill 文件夹的绝对路径。
注意事项：

metadata：必须是单行 JSON 对象（这是 OpenClaw 解析器的特殊要求）。

requires：可以用它来限制环境。例如 "bins": ["python3"] 表示只有系统安装了 Python3 时，该 Skill 才会激活。

{baseDir}：在指令中使用此占位符来引用你文件夹内的脚本。

3. 实现功能逻辑
在同级目录下创建实现逻辑的代码文件（如 main.py）。

Python
# main.py 示例
import sys
import argparse

parser = argparse.ArgumentParser()
parser.add_argument("--query", help="查询内容")
args = parser.parse_args()

print(f"收到查询：{args.query}。正在执行自定义逻辑...")
# 在这里编写你的业务逻辑
4. 让 OpenClaw 识别 Skill
如果你没有将文件夹放在默认的 ~/.openclaw/skills 下，需要修改 OpenClaw 的全局配置文件 ~/.openclaw/openclaw.json。

编辑 skills.load.extraDirs 字段：

JSON
{
  "skills": {
    "load": {
      "extraDirs": ["/你的/自定义/技能/路径"],
      "watch": true
    },
    "entries": {
      "my-custom-skill": {
        "enabled": true,
        "env": {
          "MY_API_KEY": "your_value_here"
        }
      }
    }
  }
}
watch: 设置为 true 后，你修改 SKILL.md 时 OpenClaw 会自动热重载，无需重启。

5. 验证与调用
识别验证：启动 OpenClaw 或查看日志，确认 Skill 已加载。如果是使用 macOS 版 OpenClaw 客户端，通常能在 Skills 列表中看到带你设置的图标的技能。

调用方式：

自动调用（推理）：Agent 会读取 SKILL.md 中的说明。如果用户的问题匹配了你的描述，Agent 会根据“使用方法”部分的指令，通过内置的 exec 工具自动运行你的脚本。

手动调用（斜杠命令）：如果在元数据中设置了 user-invocable: true，你可以在聊天框输入 /my-custom-skill 直接触发。

关键点总结
单行元数据：metadata 字段在 YAML 头部必须写成一行 JSON。

路径占位符：脚本调用务必使用 {baseDir} 确保路径跨平台兼容。

环境门控：利用 requires.env 或 requires.bins 确保 Skill 仅在依赖满足时才加载，避免运行时报错。
